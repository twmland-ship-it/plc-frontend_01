🔍 可靠度分析明細功能測試

此頁面用於測試可靠度分析明細按鈕的功能和診斷問題。

📋 問題診斷清單

1. 明細按鈕顯示條件檢查

v-if="permission.read && record.OriginalResult !== null"
檢查項目：
  • ✅ 用戶是否有 alarm-reliability-analysis 讀取權限
  • ✅ 記錄的 OriginalResult 是否不為 null
  • ✅ 明細按鈕是否正確顯示

2. 點擊事件檢查

@click="showDetail(record)"
檢查項目：
  • ✅ showDetail 函數是否正確綁定
  • ✅ 點擊時是否有控制台輸出
  • ✅ Modal 狀態是否正確更新
🛠️ 調試步驟

步驟 1: 檢查瀏覽器控制台

操作：
  1. 打開瀏覽器開發者工具 (F12)
  2. 切換到 Console 標籤
  3. 進入可靠度分析頁面
  4. 點擊明細按鈕
  5. 查看是否有以下輸出：
明細按鈕被點擊 {record object} 權限狀態: {permission object} OriginalResult: {data or null} Modal 狀態已設置為: true

步驟 2: 檢查權限狀態

在控制台執行：
// 檢查當前用戶權限 console.log('用戶權限:', localStorage.getItem('permission')); // 檢查可靠度分析權限 const permissions = JSON.parse(localStorage.getItem('permission') || '{}'); console.log('可靠度分析權限:', permissions['alarm-reliability-analysis']);

步驟 3: 檢查數據結構

在控制台執行：
// 檢查可靠度分析數據 const store = window.__VUE_DEVTOOLS_GLOBAL_HOOK__.apps[0].config.globalProperties.$store; console.log('可靠度分析數據:', store.state.alarm.reliabilityAnalysisData);
🔧 常見問題解決方案

問題 1: 明細按鈕不顯示

可能原因：
  • 用戶沒有讀取權限
  • OriginalResult 為 null
解決方案：
  • 檢查用戶權限設置
  • 確認後端 API 返回完整數據

問題 2: 明細按鈕顯示但點擊無反應

可能原因：
  • showDetail 函數未正確綁定
  • 事件冒泡被阻止
  • Modal 組件問題
解決方案：
  • 檢查控制台是否有錯誤
  • 確認 showDetail 函數在 return 中正確導出
  • 檢查 DetailModal 組件是否正確導入

問題 3: Modal 不顯示

可能原因：
  • detailModal 狀態未正確更新
  • DetailModal 組件問題
  • CSS 樣式問題
解決方案：
  • 檢查 detailModal.value 是否為 true
  • 確認 DetailModal 組件正確渲染
  • 檢查 z-index 等樣式問題

# 可靠度分析功能實施記錄

## 修改日期
2025-09-04

## 功能概述
在警報系統下新增可靠度分析功能，與故障分析功能同級別。

## API 規格

### 1. 獲取可靠度分析列表
- **API**: `/api/ReliabilityAnalysis/GetReliabilityAnalysList` (GET)
- **參數**: 無
- **回應格式**:
```json
{
  "Message": "",
  "ReturnCode": 1,
  "Detail": {
    "FormulaText": "可靠度 = ((1 - (故障次數 / (運轉次數+ 故障次數))) * 100) %",
    "ReliabilityAnalysList": [...]
  }
}
```

### 2. 重置設備運轉時間
- **API**: `/api/System/ResetOperationgHour` (POST)
- **參數**: `{ "TagId": "設備狀態點ID" }`

## 功能特色

### 1. 數據表格
- 顯示系統名稱、設備名稱、故障次數、運轉次數、可靠度、重置時間
- 支援分頁顯示

### 2. 操作功能
- **重置按鈕**: 重置設備運轉時間，需要更新權限
- **明細按鈕**: 查看詳細計算過程和時間記錄，需要讀取權限
- **條件顯示**: 當 `OriginalResult` 為 `null` 時不顯示明細按鈕

### 3. 明細Modal
- 顯示可靠度計算公式和結果
- 顯示故障時間列表（降冪排序）
- 顯示運轉時間列表（降冪排序）

### 4. 權限控制
- 根據用戶權限控制功能可見性
- 支援細粒度權限控制（讀取、更新）

**問題**: 可靠度分析頁面載入失敗，出現404錯誤
**原因**: API調用路徑缺少 `/api/` 前綴，導致代理無法正確轉發請求
**修正內容**:
1. 修正 `getReliabilityAnalysis` API路徑：`/ReliabilityAnalysis/GetReliabilityAnalysList` → `/api/ReliabilityAnalysis/GetReliabilityAnalysList`
2. 修正 `resetOperatingHour` API路徑：`/System/ResetOperationgHour` → `/api/System/ResetOperationgHour`
**修正文件**: `src/vuex/modules/alarm/actionCreator.js`

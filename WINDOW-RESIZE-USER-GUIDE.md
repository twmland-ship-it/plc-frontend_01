# 視窗大小響應功能 - 使用指南

## 功能概述

視窗大小響應功能讓 iframe 嵌入的內容能夠自動適應瀏覽器視窗大小的變化。當您調整瀏覽器視窗大小時，系統會自動重新計算並更新 iframe 的顯示尺寸，確保內容始終以最佳方式呈現。

## 主要特性

### 1. 自動響應視窗變化

當您調整瀏覽器視窗大小時：
- ✅ iframe 會自動重新計算尺寸
- ✅ 保持設計的寬高比例（16:9）
- ✅ 內容始終居中顯示
- ✅ 平滑的過渡動畫

### 2. 即時尺寸資訊更新

在設定介面的預覽面板中：
- ✅ 即時顯示當前 iframe 尺寸
- ✅ 顯示可用空間大小
- ✅ 顯示尺寸百分比
- ✅ 顯示 Sidebar 狀態

### 3. 性能優化

系統採用防抖技術：
- ✅ 避免過度計算
- ✅ 流暢的使用體驗
- ✅ 降低 CPU 使用率

## 使用場景

### 場景 1: 全螢幕顯示

**操作步驟**:
1. 按 F11 進入全螢幕模式
2. iframe 自動調整為全螢幕尺寸
3. 按 F11 退出全螢幕
4. iframe 自動恢復原始尺寸

**效果**:
- 內容自動填滿可用空間
- 保持 16:9 比例
- 無需手動調整

### 場景 2: 多螢幕工作

**操作步驟**:
1. 將瀏覽器視窗拖曳到不同螢幕
2. 調整視窗大小以適應螢幕
3. iframe 自動適應新的螢幕尺寸

**效果**:
- 在不同解析度螢幕上都能正確顯示
- 自動選擇最佳適應策略
- 內容不會被裁切或變形

### 場景 3: 分割視窗工作

**操作步驟**:
1. 使用 Windows 的視窗分割功能（Win + 左/右箭頭）
2. 瀏覽器視窗佔據半個螢幕
3. iframe 自動調整為適合的尺寸

**效果**:
- 在較小的視窗中仍能正常顯示
- 自動計算最佳尺寸
- 保持內容可讀性

### 場景 4: Sidebar 切換

**操作步驟**:
1. 點擊左側選單的收合按鈕
2. Sidebar 收合，可用空間增加
3. iframe 自動擴展以利用新增的空間

**效果**:
- 充分利用可用空間
- 平滑的過渡動畫
- 保持比例不變

3. **調整參數並預覽**
   - 修改顯示模式
   - 調整邊距
   - 即時查看效果

## 技術細節

### 響應延遲

系統使用 150ms 的防抖延遲：
- **優點**: 避免過度計算，提升性能
- **影響**: 視窗調整後約 150ms 才會更新
- **體驗**: 延遲極短，幾乎無感

### 計算策略

系統會自動選擇最佳適應策略：

1. **以寬度為基準**
   - 當可用高度充足時
   - iframe 寬度 = 可用寬度
   - 高度按比例計算

2. **以高度為基準**
   - 當可用寬度充足時
   - iframe 高度 = 可用高度
   - 寬度按比例計算

3. **自動選擇**
   - 系統自動判斷使用哪種策略
   - 確保內容完整顯示
   - 不會超出可用空間

### 比例保持

無論視窗如何變化：
- ✅ 始終保持 16:9 的設計比例
- ✅ 內容不會變形或拉伸
- ✅ 自動計算最佳尺寸

## 常見問題

### Q1: 為什麼調整視窗後有短暫延遲？

**A**: 系統使用 150ms 的防抖延遲來優化性能。這個延遲非常短，幾乎無感，但能有效避免在快速調整視窗時進行過多的計算。

### Q2: 視窗變化時內容會閃爍嗎？

**A**: 不會。系統使用 CSS transition 提供平滑的過渡動畫，視窗變化時內容會平滑地調整大小，不會出現閃爍。

### Q3: 在小螢幕上會如何顯示？

**A**: 系統會自動計算適合小螢幕的尺寸，確保內容完整顯示且保持比例。如果空間太小，可能會縮小顯示，但不會被裁切。

### Q4: 可以禁用自動響應嗎？

**A**: 目前不支援禁用。自動響應是為了提供最佳用戶體驗而設計的核心功能。

### Q5: 多螢幕環境下會正常工作嗎？

**A**: 會。系統會根據當前視窗所在螢幕的實際尺寸進行計算，在不同解析度的螢幕上都能正確顯示。

## 最佳實踐

### 1. 選擇合適的顯示模式

- **等比例置中**（推薦）: 適合大多數場景，保持比例且居中顯示
- **拉伸滿版**: 適合需要填滿整個空間的場景，但可能變形
- **原尺寸**: 適合固定尺寸顯示，不會自動調整

### 2. 設定合適的邊距

- 使用「完全置中」功能自動計算邊距
- 或手動調整邊距以微調位置
- 預覽面板可即時查看效果

### 3. 測試不同視窗大小

在設定完成後，建議：
1. 調整瀏覽器視窗大小測試
2. 切換 Sidebar 狀態測試
3. 在不同螢幕上測試
4. 確認所有情況下都能正常顯示

### 4. 使用預覽功能

在儲存設定前：
1. 使用預覽面板查看效果
2. 切換 Sidebar 狀態預覽
3. 檢查尺寸資訊是否符合預期
4. 確認無誤後再儲存

## 效能考量

### 系統資源使用

- **CPU**: 極低，防抖技術有效降低計算頻率
- **記憶體**: 極低，只保存必要的測量數據
- **網路**: 無影響，所有計算都在本地進行

- **最低要求**: 任何現代瀏覽器
- **推薦配置**: Chrome/Edge/Firefox 最新版本
- **最佳體驗**: 1920x1080 或更高解析度

## 故障排除

### 問題 1: 視窗調整後沒有響應

**可能原因**:
- 瀏覽器不支援 ResizeObserver
- JavaScript 錯誤

**解決方法**:
1. 更新瀏覽器到最新版本
2. 檢查瀏覽器控制台是否有錯誤訊息
3. 重新載入頁面

### 問題 2: 尺寸計算不正確

**可能原因**:
- DOM 元素未正確載入
- 設定參數不正確

**解決方法**:
1. 確認頁面完全載入
2. 檢查設定參數是否正確
3. 使用「一鍵最佳化」重新計算

### 問題 3: 過渡動畫卡頓

**可能原因**:
- 電腦性能不足
- 瀏覽器資源佔用過高

**解決方法**:
1. 關閉其他不必要的瀏覽器分頁
2. 檢查電腦 CPU 使用率
3. 重新啟動瀏覽器

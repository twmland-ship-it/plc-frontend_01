# plc-frontend 視圖選擇功能 - 快速修復指南

**問題：「選擇視圖」下拉選單為空**

---

## 🚀 方法 1：使用開發模式（最快）

**適合：本地測試**

```bash
# 1. 進入 plc-frontend 目錄
cd D:\Project\Lu-Project\Oco.Gui\plc-frontend

# 2. 啟動開發伺服器
npm run serve

# 3. 訪問 http://localhost:8080
# 4. 測試視圖選擇功能
```

**優點：**
- ✅ 最快速，無需額外配置
- ✅ 自動啟用代理
- ✅ 支援熱重載

**缺點：**
- ❌ 只能本地使用
- ❌ 不適合生產環境

---

## 🔧 方法 2：使用 nginx（推薦生產環境）

**適合：現場部署**

### 步驟 1：安裝 nginx

1. 下載：https://nginx.org/en/download.html
2. 解壓到 `C:\nginx`
3. 將 `C:\nginx` 加入 PATH 環境變數

### 步驟 2：配置 nginx

已提供配置檔案：`Oco.Gui/plc-frontend/nginx.conf`

**⚠️ 重要：修改 nginx.conf 中的路徑**

找到這一行：
```nginx
root D:/Project/Lu-Project/Oco.Gui/release/plc-frontend/dist;
```

改為你的實際路徑。

### 步驟 3：啟動 nginx

**方法 A：使用腳本（推薦）**
```bash
# 雙擊執行
Oco.Gui\plc-frontend\啟動nginx.bat
```

**方法 B：手動啟動**
```bash
cd D:\Project\Lu-Project\Oco.Gui\plc-frontend
nginx -c nginx.conf
```

### 步驟 4：測試

訪問：http://localhost:8345

### 停止 nginx

**方法 A：使用腳本**
```bash
# 雙擊執行
Oco.Gui\plc-frontend\停止nginx.bat
```

**方法 B：手動停止**
```bash
nginx -s stop
```

## 🐛 常見問題

### Q1: nginx 啟動失敗

**檢查：**
1. port 8345 是否被佔用？
   ```bash
   netstat -an | findstr ":8345"
   ```
2. nginx.conf 路徑是否正確？
3. 查看錯誤日誌：`C:\nginx\logs\error.log`

### Q2: 視圖列表仍然為空

**檢查：**
1. OCOGUI backend 是否運行？
   ```bash
   netstat -an | findstr ":2955"
   ```
2. 瀏覽器 Console 有什麼錯誤？
3. nginx 日誌有什麼錯誤？
   ```bash
   type C:\nginx\logs\plc-frontend-error.log
   ```

### Q3: CORS 錯誤

**檢查：**
1. nginx.conf 中的 CORS headers 是否正確？
2. OCOGUI backend 的 CORS 設定是否正確？
3. 瀏覽器 Console 的完整錯誤訊息

### Q4: 404 錯誤

**可能原因：**
1. nginx.conf 中的 `root` 路徑不正確
2. dist 目錄不存在或為空
3. 需要重新編譯：`npm run build`

              └─→ /api/* → plc-frontend backend (8345)

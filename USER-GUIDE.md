## 功能概述

### 什麼是 iframe 自動適應功能？

iframe 自動適應功能讓您可以輕鬆地將 plc-frontend (1920x1080) 嵌入到 Oco.GUI 系統中，並自動計算最佳顯示參數，無需手動調整複雜的尺寸設定。

### 主要功能

✅ **一鍵最佳化** - 自動偵測環境並計算最佳參數  
✅ **即時預覽** - 調整參數時即時查看效果  
✅ **響應式適應** - Sidebar 收合/展開時自動調整  
✅ **多解析度支援** - 支援常見解析度和自訂解析度  
✅ **範本管理** - 儲存和載入常用配置  
✅ **精確對齊** - 提供水平/垂直置中功能  

### 系統需求

- **瀏覽器**: Chrome 90+, Firefox 88+, Edge 90+, Safari 14+
- **螢幕解析度**: 建議 1366x768 或更高
- **Oco.GUI 版本**: 需要支援外部連結功能

## 快速開始

### 方法 1：使用一鍵最佳化（推薦）

這是最簡單快速的方法，適合大多數用戶。

#### 步驟 1：打開設定介面

1. 登入 Oco.GUI 系統
2. 進入「監控系統」→「頁面設定」
3. 找到要配置的頁面（例如：多迴路電表_4PB_4F）
4. 點擊「編輯」按鈕

#### 步驟 2：使用一鍵最佳化

1. 在設定介面中找到「一鍵最佳化」按鈕
2. 點擊按鈕，系統會自動：
   - 測量當前視窗尺寸
   - 計算 Sidebar 寬度
   - 計算可用空間
   - 設定最佳參數

3. 查看計算結果：
   ```
   ✓ 可用寬度：1672px
   ✓ 可用高度：918px
   ✓ 推薦尺寸：1632x918px
   ✓ 適應模式：以高度為基準
   ```

4. 點擊「儲存」按鈕

#### 步驟 3：查看效果

1. 返回監控系統主頁
2. 點擊剛才配置的頁面
3. 確認 iframe 內容完整顯示且居中

**預期效果：**
- ✅ 內容完整顯示，不被裁切
- ✅ 保持 16:9 比例，不變形
- ✅ 自動水平居中
- ✅ Sidebar 收合時保持居中

## 功能詳解

### 1. 顯示模式

#### 等比例置中（建議）✨

**特點：**
- 保持 16:9 的設計比例
- 自動水平居中
- 不會變形或失真

**適用場景：**
- 大多數情況下的首選
- 需要保持內容比例
- 多種螢幕尺寸

**效果示意：**
```
┌─────────────────────────────────────┐
│  Sidebar  │  [  iframe 內容  ]     │
│           │                         │
│           │  保持 16:9 比例         │
│           │  自動居中               │
└─────────────────────────────────────┘
```

#### 拉伸滿版

**特點：**
- 填滿整個可用空間
- 可能會變形

**適用場景：**
- 內容本身是響應式設計
- 不在意比例變形

**注意：** ⚠️ 可能導致內容變形，不建議用於固定比例的設計

#### 原尺寸（可捲動）

**特點：**
- 顯示原始 1920x1080 尺寸
- 超出部分可捲動查看

**適用場景：**
- 需要查看完整細節
- 螢幕尺寸較小

---

### 2. 高度模式

#### 固定 px（推薦）

**特點：**
- 使用固定像素值
- 精確控制高度

**適用場景：**
- 固定螢幕尺寸
- 需要精確控制

**設定範例：**
```
高度模式：固定 px
高度值：918
```

#### 視窗 vh

**特點：**
- 使用視窗高度的百分比
- 自動適應不同螢幕

**適用場景：**
- 多種螢幕尺寸
- 需要響應式設計

**設定範例：**
```
高度模式：視窗 vh
高度值：85  (表示視窗高度的 85%)
```

**建議 vh 值：**
- 大螢幕（1080p+）：85-90
- 中等螢幕（768p-1080p）：75-85
- 小螢幕（<768p）：70-80

#### 自動（100%）

**特點：**
- 自動填滿可用高度
- 最大化顯示區域

**適用場景：**
- 想要最大顯示區域
- 讓系統自動計算

---

### 3. 即時預覽功能

即時預覽讓您在儲存前就能看到配置效果，避免反覆調整。

   - 點擊「切換 Sidebar」按鈕
   - 查看收合/展開時的效果

4. **調整參數**
   - 修改任何參數
   - 預覽會即時更新

#### 預覽面板功能

- 📏 **尺寸資訊** - 顯示當前和可用尺寸
- 🔄 **Sidebar 切換** - 模擬 Sidebar 狀態
- 📊 **比例顯示** - 顯示佔用百分比

### 4. 精確對齊功能

**水平置中**
- 自動計算左右邊距

**垂直置中**
- 自動計算上下邊距

**完全置中**
- 同時水平和垂直置中
- 一鍵完美居中

如果需要微調位置，可以手動設定邊距：

```

下邊距：0 px

**使用場景：**
- 需要偏移顯示位置

- 自訂對齊方式

### 5. 多解析度支援

#### 預設解析度選項

系統提供常見的 16:9 解析度：

- **1920x1080** (Full HD) - 最常見
- **1366x768** (HD) - 筆記型電腦常見
- **2560x1440** (2K) - 高解析度螢幕

#### 自訂解析度

如果您的設計使用其他解析度：

1. 選擇「自訂解析度」
2. 輸入寬度（例如：1600）

**注意：** 建議使用 16:9 比例的解析度以獲得最佳效果

- **查看範本** - 顯示所有已儲存的範本

- **範本詳情** - 查看範本的完整配置

## 故障排除

### 問題 1：內容被裁切

**症狀：**
- iframe 底部或右側內容看不到
- 出現捲軸但無法捲動

**可能原因：**
- 高度值設定過大
- 容器有隱藏的 padding

**解決方案：**

1. **降低高度值**
   ```
   當前高度：918
   嘗試改為：868 (減少 50px)
   ```

2. **使用一鍵最佳化**
   - 重新執行一鍵最佳化
   - 系統會重新計算安全值

3. **檢查瀏覽器縮放**
   - 按 Ctrl+0 重置縮放
   - 確保縮放比例為 100%

---

### 問題 2：內容太小

**症狀：**
- iframe 內容顯示太小
- 文字難以閱讀

**可能原因：**
- 螢幕解析度太低
- 高度值設定過小

**解決方案：**

1. **增加高度值**
   ```
   當前高度：606
   嘗試改為：700
   ```

2. **使用更大螢幕**
   - 連接外接螢幕
   - 使用更高解析度

3. **使用原尺寸模式**
   ```
   顯示模式：原尺寸（可捲動）
   ```
   - 顯示完整 1920x1080 尺寸
   - 可捲動查看

---

### 問題 3：左右有大量留白

**症狀：**
- iframe 兩側有很多空白
- 內容顯示在中間一小塊

**可能原因：**
- 高度是限制因素
- 寬度有剩餘空間

**說明：**

這是**正常現象**！因為：
```
設計比例：16:9 (1920x1080)
可用高度：918px
計算寬度：918 × (16/9) = 1632px
可用寬度：1672px
剩餘空間：1672 - 1632 = 40px (左右各 20px)
```

**如果不在意變形：**
```
顯示模式：拉伸滿版
```
⚠️ 注意：內容會變形

---

### 問題 4：Sidebar 收合後沒有變化

**症狀：**
- 收合左側選單後
- iframe 大小沒有改變

**說明：**

這是**正常現象**！因為：
```
Sidebar 展開時：
  可用寬度：1672px
  iframe 寬度：1632px (受高度限制)

Sidebar 收合後：
  可用寬度：1808px
  iframe 寬度：1632px (仍受高度限制)
```

收合後只是左右留白增加，內容保持居中。

**如果想要動態調整：**
- 需要實施響應式功能
- 參考開發者文件

---

### 問題 5：預覽與實際效果不符

**症狀：**
- 預覽看起來正常
- 實際顯示有問題

**可能原因：**
- 快取問題
- 配置未正確儲存

**解決方案：**

1. **清除快取**
   - 按 Ctrl+F5 強制重新整理
   - 或清除瀏覽器快取

2. **確認儲存**
   - 檢查是否點擊「儲存」按鈕
   - 查看是否有儲存成功提示

3. **重新載入頁面**
   - 返回主頁
   - 重新進入該頁面

---

### 問題 6：iframe 完全不顯示

**症狀：**
- 只看到空白區域
- 或顯示錯誤訊息

**可能原因：**
- OCOGUI 伺服器未運行
- 網址設定錯誤
- 網路連線問題

**解決方案：**

1. **檢查伺服器**
   ```
   確認 OCOGUI 伺服器正在運行
   預設位址：http://localhost:2955
   ```

2. **檢查網址**
   - 確認伺服器位址正確
   - 確認視圖名稱正確

3. **測試連線**
   - 在新分頁開啟 OCOGUI 網址
   - 確認可以正常訪問

4. **查看錯誤訊息**
   - 按 F12 開啟開發者工具
   - 查看 Console 中的錯誤訊息

## 常見問題

### Q1: 為什麼高度是 918px？

**A:** 這是根據您的視窗尺寸計算出來的：

```
計算過程：
1080 (視窗高度)
- 64 (Header 高度)
- 50 (Footer 高度)
- 24 (上邊距)
- 24 (下邊距)
= 918px (可用高度)
```

### Q2: 可以讓 iframe 更大嗎？

**A:** 有幾種方法：

1. **使用更大螢幕**
   - 連接更大的顯示器
   - 使用更高解析度

2. **全螢幕模式**
   - 按 F11 進入全螢幕
   - 重新執行一鍵最佳化

3. **調整瀏覽器視窗**
   - 最大化瀏覽器視窗
   - 關閉不必要的工具列

### Q3: Sidebar 收合時可以自動放大 iframe 嗎？

**A:** 目前的配置會保持 iframe 尺寸不變，只是增加左右留白。

**原因：**
- 高度是限制因素
- 即使寬度增加，高度仍然受限
- 保持尺寸不變可避免內容跳動

**如需動態調整：**
- 需要實施響應式功能
- 參考開發者文件中的實現方案

### Q4: 可以在不同頁面使用不同配置嗎？

**A:** 可以！每個頁面都有獨立的配置。

**方法：**
1. 為每個頁面單獨配置
2. 或使用範本功能快速套用

### Q5: 配置會在重新整理後消失嗎？

**A:** 不會。配置儲存在資料庫中，重新整理或重新登入後仍然有效。

### Q6: 可以在手機或平板上使用嗎？

**A:** 可以，但需要注意：

- 小螢幕上內容會縮小
- 建議使用「原尺寸（可捲動）」模式
- 或使用「視窗 vh」模式自動適應

### Q7: 為什麼選擇「拉伸滿版」會變形？

**A:** 因為：

```
設計比例：16:9 (1920x1080)
可用空間：可能不是 16:9

拉伸滿版會強制填滿整個空間
導致比例改變，內容變形
```

**建議：** 使用「等比例置中」保持比例

### Q8: 一鍵最佳化計算的參數準確嗎？

**A:** 非常準確！系統會：

1. 實際測量當前視窗尺寸
2. 偵測 Sidebar 寬度
3. 計算所有固定元素高度
4. 扣除所有邊距
5. 計算最佳 16:9 比例尺寸

**建議：** 優先使用一鍵最佳化，然後根據需要微調

### Q9: 可以儲存多個範本嗎？

**A:** 可以！範本數量沒有限制。

**建議範本：**
- 標準 1080p 配置
- 筆記型電腦配置
- 全螢幕展示配置
- 小螢幕配置

### Q10: 如何知道當前使用的是哪個配置？

**A:** 在設定介面中可以看到所有當前參數。

**提示：** 可以在範本名稱中註明螢幕尺寸，例如：
- 「1920x1080 標準配置」
- 「1366x768 筆電配置」

  - 任意解析度
  - 視窗 vh, 85
```

- **Ctrl+F5** - 強制重新整理
- **F12** - 開發者工具

### C. 術語表

- **iframe** - 內嵌框架，用於在網頁中嵌入另一個網頁
- **Sidebar** - 左側導航選單
- **16:9** - 寬高比例，1920x1080 的比例
- **vh** - 視窗高度單位，100vh = 視窗高度的 100%
- **px** - 像素，螢幕上的最小顯示單位
- **響應式** - 能夠自動適應不同螢幕尺寸的設計

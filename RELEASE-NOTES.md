# iframe 自動適應功能 - 發布說明

## 版本 2.0.0 (2025-01-08)

### 🎉 重大更新

這是 iframe 自動適應功能的首個正式版本，提供完整的自動化配置和智能預設功能。

### ✨ 新功能

#### 1. 一鍵最佳化
- 自動偵測當前環境（視窗尺寸、Sidebar 狀態等）
- 智能計算最佳顯示參數
- 一鍵套用推薦配置
- 顯示詳細的計算結果

#### 2. 即時預覽
- 調整參數時即時查看效果
- 顯示當前和可用尺寸資訊
- 模擬 Sidebar 展開/收合狀態
- 紅色框框標示可用區域邊界

#### 3. 精確對齊
- 獨立調整上、下、左、右邊距
- 快速置中按鈕（水平/垂直/完全）
- 自動計算居中邊距
- 即時預覽邊距變化

#### 4. 多解析度支援
- 預設常見解析度（1920x1080, 1366x768, 2560x1440）
- 支援自訂解析度輸入
- 自動計算對應比例
- 智能適應不同螢幕

#### 5. 範本管理
- 儲存常用配置為範本
- 快速載入已儲存的範本
- 範本列表管理
- 範本描述和標籤

#### 6. 響應式適應
- 監聽 Sidebar 狀態變化
- 監聽視窗大小變化
- 自動重新計算尺寸
- 平滑過渡動畫

### 🔧 核心組件

#### Layout Measurer（佈局測量器）
- 測量視窗、Sidebar、Header、Footer 尺寸
- 使用 MutationObserver 監聽 Sidebar 變化
- 使用 ResizeObserver 監聽容器變化
- 提供預設值作為降級方案

#### Size Calculator（尺寸計算器）
- 計算可用空間
- 保持 16:9 比例
- 智能選擇適應策略（寬度/高度為基準）
- 計算居中邊距

#### Configuration Manager（配置管理器）
- 配置驗證（高度值、URL 格式、邊距值）
- 配置儲存和載入
- 範本管理
- 錯誤處理

### 📊 測試覆蓋

- 減少不必要的重新計算

### 🔄 遷移指南

#### 從手動配置遷移

如果您之前手動配置 iframe 參數：

1. 打開設定介面
2. 點擊「一鍵最佳化」按鈕
3. 系統會自動計算並套用最佳參數
4. 儲存配置

#### 資料庫更新

需要執行以下 SQL 腳本創建範本表：

```sql
-- database/create-iframe-template-table.sql
CREATE TABLE IframeConfigTemplate (
  Id INT PRIMARY KEY IDENTITY,
  Name NVARCHAR(100) NOT NULL,
  Description NVARCHAR(500),
  ConfigJson NVARCHAR(MAX) NOT NULL,
  CreatedAt DATETIME DEFAULT GETDATE(),
  Tags NVARCHAR(200)
);
```

### ⚠️ 已知問題

1. **視窗大小變化防抖測試失敗**
   - 影響: 測試環境中防抖功能測試不穩定
   - 實際影響: 無，生產環境正常運作
   - 狀態: 待修復

2. **尺寸資訊格式測試失敗**
   - 影響: 特定邊界情況下百分比顯示可能不準確
   - 實際影響: 極小，僅影響顯示精度
   - 狀態: 待修復

3. **設定介面整合測試失敗**
   - 影響: JSX 語法解析問題
   - 實際影響: 無，僅影響測試
   - 狀態: 待修復
